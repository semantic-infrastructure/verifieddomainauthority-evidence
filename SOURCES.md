## Primary sources (anchors)

The following sources serve as **definitional and technical anchors** for
domain control, publication integrity, and verifiable proof mechanisms.

1. **IETF RFC 1034 — Domain Names: Concepts and Facilities**  
   https://www.rfc-editor.org/rfc/rfc1034.html

2. **IETF RFC 1035 — Domain Names: Implementation and Specification**  
   https://www.rfc-editor.org/rfc/rfc1035.html

3. **IETF RFC 5280 — Internet X.509 Public Key Infrastructure Certificate and CRL Profile**  
   https://www.rfc-editor.org/rfc/rfc5280.html

4. **IETF RFC 6962 — Certificate Transparency**  
   https://www.rfc-editor.org/rfc/rfc6962.html

5. **IETF RFC 3161 — Time-Stamp Protocol (TSP)**  
   https://www.rfc-editor.org/rfc/rfc3161.html

6. **W3C PROV — Provenance Data Model (Overview)**  
   https://www.w3.org/TR/prov-overview/

7. **W3C Verifiable Credentials Data Model 2.0**  
   https://www.w3.org/TR/vc-data-model-2.0/

## Citation discipline

- Claims of authority must reference at least one **primary source**.  
- Context sources may support explanation but must not redefine scope.  
- No restricted or paywalled text is reproduced.  
- URLs are chosen for **canonical status and long-term stability**.
